# GP-Chat システム仕様書 (software-sheet.md)

本ドキュメントは、GCP (Vertex AI) および Azure OpenAI などの大規模言語モデル（LLM）APIを活用した、Streamlitベースの多機能チャットアプリケーション「GP-Chat」の仕様書です。

---

## 1. システム概要
GP-Chatは、対話型AIチャットを基盤とし、マルチモーダルなドキュメント分析、ローカル実行エンジンによるデータ分析とグラフ自動描画、およびヘッドレスブラウザ連携によるHTML/PDFレポートスライド自動生成機能を備えた、高度なアシスタントアプリケーションです。GCP Vertex AIのAPIを主系としつつ、自動フォールバック先および特定コーディングモデル（`gpt-5.3-codex`）の直接接続先としてAzure OpenAIを統合し、柔軟かつ堅牢な稼働安定性を有しています。

---

## 2. システムアーキテクチャとファイル構成
本システムのプログラムは主に `src/gp_chat` ディレクトリ内に配置されています。各モジュールの役割は以下の通りです。

| 分類 | ファイル名 | 役割・説明 |
| :--- | :--- | :--- |
| **起動・エントリー** | `main_runner.py` | アプリケーションのエントリーポイント。`main.py` の実行を呼び出す。 |
| | `main.py` | メインロジック。画面制御、チャット描画、セッション連携、モデルIDに応じたGCPバイパスやデプロイメント切替。 |
| **ユーザーインターフェース** | `sidebar.py` | UIサイドバー領域の描画。モデル、思考レベル、Canvas、ファイル添付、履歴の制御。 |
| **設定・テキスト定数** | `config.py` | デフォルトのセッション情報、UI表示テキスト、選択可能なモデル（`gpt-5.3-codex` を含む）、定数。 |
| | `config.yaml` | Canvasへのファイルロード時に許可する拡張子リストなどの設定ファイル。 |
| | `prompts.yaml` | デフォルトのプロンプト定義。起動時にプロジェクトルート直下の `prompts/prompts.yaml` に自動コピーされる。 |
| **AIパース・共通処理** | `utils.py` | Word・Excel・PPT・PDF等のファイルパース処理、プロンプトのロード（自動コピー・書き込み含む）、Gemini用コンテキストの構築。 |
| **状態・データ管理** | `state_manager.py` | 会話履歴の自動保存、手動再開、中断リカバリー、システムログ（デバッグログ）の収集と表示。 |
| | `data_manager.py` | データ分析時に生成される一時ファイルやアップロードファイルをセッションIDごとに管理。 |
| **LLMルーティング** | `llm_router.py` | Vertex AI クライアントの初期化。標準/優先ルート制御、API 429等のエラー発生時の自動リトライ。 |
| **特化型エージェント** | `code_agent.py` | AIが生成したコードの実行を監視し、エラー時にAIへ自己修復を依頼するループ制御。 |
| | `execution_engine.py` | セッション内スコープにおけるPythonコードの実行、標準出力の回収、描画されたグラフ（matplotlib）のキャプチャ。 |
| | `reasoning_agent.py` | 「アプローチ立案 → 自己批判(Critique) → 最終結論」を行う思考プロセス特化エージェント。 |
| | `research_agent.py` | 情報過不足を評価しながらGoogle検索を自律反復する徹底調査（ReAct）エージェント。 |
| | `report_agent.py` | 会話履歴からHTMLプレゼンを構築し、ヘッドレスブラウザでPDFに自動変換して保存するエージェント。 |
| **Azure連携 (副系・直接接続)** | `azure_runtime.py` | Azure OpenAI クライアントの初期化とランタイム管理（Codex専用デプロイメント等のロード）。 |
| | `azure_context_builder.py` | Azure APIに適した形式のコンテキストオブジェクト（メッセージ履歴、添付ファイル）の構築。 |
| | `azure_supervisor_helpers.py` | GCP側のエラー状態を検知し、Azure側への切り替え要否を判定するスーパーバイザヘルパー。 |
| | `azure_*_agent.py` | 主系に対応するAzure用の各種特化型エージェント（Code, Reasoning, Research, Report, Normal）。 |
| | `azure_fault_injection.py` | Azure fallback挙動をテストするための疑似エラー注入用モジュール。 |
| | `azure_common_types.py` | Azure環境下で共通して利用されるデータクラスの定義。 |

## 3. 主要機能の仕様詳細

### ① AIモデル選択と推論レベル制御
* **モデル選択**: Gemini 3.5 FlashやGemini 3.1 Proなど、UIから任意のモデルを選択し、セッション中に動的に変更可能。
* **`gpt-5.3-codex` 直接接続**: モデル一覧より `gpt-5.3-codex` を選択した場合、通常のGCP Vertex AIへの呼び出し処理をスキップ（バイパス）し、最初から直接 Azure OpenAI に接続して応答を生成します。
* **思考レベル（Thinking Level / reasoning_effort）**:
  * `high`: 標準的な推論。
  * `low`: 思考プロセスの簡略化による高速な応答。
  * `deep`: 推論特化モード。`reasoning_agent.py` によって「多角的なアプローチ立案」「自己批判 (Critique)」「結果の統合」の3段階を巡る独自のマルチターン・ワークフローを実行し、自己省察を含めた深い推論出力を生成。
* **Google検索（Grounding）のトグル**: ONにすると、Gemini APIが自動的にGoogle検索結果をグラウンディングソースとして活用。検索クエリとソースURLをUI上にアコーディオンで表示。

### ② コードエディタ（Canvas）
* **マルチエディタ構成（常時有効化）**: 起動時および履歴復元時から常に有効であり、最大40個までの独立したPythonコード編集エリア（`st_ace` 使用）を「Canvasを追加」ボタンによって追加・保持可能。
* **対話型コード処理**: 
  * 「AIへ送信」がONのCanvasデータは、チャット送信時にコードスニペットとしてプロンプトに自動挿入。なお、1番目のCanvasは何も入力されていない起動時は「AIへ送信」がOFF(False)になりますが、テキストの入力やファイルのロードが行われると自動的にONになります。
  * 「レビュー」ボタンにより、AIがそのコードの改善案を提案。
  * 「検証」ボタンでローカル環境下の `pylint` を実行し、指摘結果をプロンプト化してAIがエラー修正案を作成する。
  * 外部ファイル（`.py` など）をドラッグ＆ドロップでCanvasに直接ロード可能。

### ③ データ分析とグラフ描画（Auto-Plot）
* **コード自動実行**: チャット内でAIが生成した ` ```python ` ブロックを検知し、ローカル環境で自動実行。
* **グラフキャプチャ**: 実行結果として描画されたグラフ（`matplotlib` / `seaborn`）をキャプチャし、Base64化してUIチャット欄に画像としてインライン表示。
* **日本語フォント文字化け防止**: OS（Windows / macOS / Linux）を自動判定し、最適な日本語フォント（Meiryo、Hiragino Sans、Noto Sans CJK JP等）を動的に設定。
* **自己修復ループ（Auto-Fix）**:
  AIが生成したプログラムにバグがありエラー（トレースバック）が発生した場合、実行エンジンがそれを検知。エラー文をプロンプトとしてAIに返し、最大2回まで自動で修正・再実行を試みる（`code_agent.py`）。

### ④ マルチモーダルファイル添付
ユーザーはチャット入力とは別に、複数のファイルをアップロードしてAIに読み込ませることができます。拡張子に応じてパースされ、プロンプトの先端にコンテキストとして注入されます。

* **画像形式** (`.png`, `.jpg`, `.jpeg`, `.bmp`, `.gif`): Gemini APIに画像データとして直接送信。
* **PDF** (`.pdf`): Gemini APIにドキュメントとして直接送信。
* **Word** (`.docx`): `python-docx` で全段落のテキストを抽出し、テキストスニペットとして送信。
* **PowerPoint** (`.ppt`, `.pptx`): Windows環境下で `pywin32` (COMオブジェクト) と連携し、各スライドを画像（PNG）に一度エクスポートした上で、画像データ群として送信。
* **Excel** (`.xlsx`, `.xlsm`, `.xls`):
  `pandas` と各エンジン（`openpyxl`, `python-calamine`, `xlrd`）を用いて全シートを読み込み、**Markdownテーブル形式**（`tabulate` 不在時はCSV形式）に自動変換して送信。マクロ有効ファイル（`.xlsm`）のデータや古い形式（`.xls`）も安全にサポート。
* **テキスト形式** (`.txt`, `.md`, `.py`, `.js`, `.json`, `.csv`, `.yaml` 等): UTF-8（失敗時はCP932/Shift-JIS）でデコードし、テキストとして送信。

### ⑤ 徹底調査モード（More Research）
* Web検索（Google Search Grounding）をバックグラウンドで反復実行し、自問自答しながら情報を多角的に集める自律エージェント型（ReAct型）モード。
* 最大3サイクル走り、毎サイクルごとに「現在の知識でユーザーの要求に答えられるか」「次に検索すべき情報は何か」をJSONで自己評価する。十分と判定されるか上限に達すると、集まった全データを統合して最終回答を出力する。

### ⑥ レポート自動生成機能（HTML/PDFスライド）
* これまでの議論履歴を要約し、AIがプレゼンテーション資料としてそのまま使える美しいHTML/CSSドキュメント（スライドデッキ）を自動生成。
* 保存先は `slide_data/<チャットタイトル>/` ディレクトリ。
* 生成されたHTMLファイルを、ローカルのヘッドレスブラウザ（Microsoft Edge または Google Chrome）の印刷機能を利用して、PDF（`A4横向きサイズ` 等）に自動エクスポートして保存する。

### ⑦ 高可用性二重化と直接接続（Azure Fallback & Direct Connection）
* **自動フォールバック**: GCP (Vertex AI) に対するAPIリクエストがエラー（レート制限の429やタイムアウトなど）に遭遇した際、自動でAzure OpenAI環境に切り替えて同じ会話とコンテキストを再送します。
* **Azure直接接続**: コーディング特化モデル（`gpt-5.3-codex`）選択時は、GCPを最初から完全にスキップ（バイパス）して直接 Azure OpenAI の該当デプロイメントに接続し応答を生成します。このとき、UIのローディング表示もAzure接続用に動的に切り替わります。
* **GCP Cloud Logging 送信の自動スキップ**: Azure OpenAI を経由して応答を生成する際（直接接続時および自動フォールバック時の両方）、GCPの Cloud Logging へのログ送信処理を自動的にスキップします。これにより、GCPの権限不足（403 Forbidden）による無駄なエラーログの発生を防ぎます。
* **コンテキスト変換**: 主系と副系でAPIのインターフェースやコンテキスト表現が異なるため、`azure_context_builder.py` を経由してAzure向けの専用コンテキスト（`input_text`, `input_image` 等の構造化辞書）に透過的に変換して処理します。

### ⑧ システムプロンプト（AIの役割）の複数選択・保存
* **プリセット選択**: 起動時にルート直下の `prompts/prompts.yaml` に定義されたプリセット（`system_presets`）からAIのシステムプロンプト（役割）を選択可能。
* **プロンプト内容の編集と保存**: プリセットの内容をテキストエリアで編集可能。新規作成も選択可能。右下の「保存するプロンプト名」は誤操作防止のためデフォルトでは常に空欄で表示される。
* **「このまま実行(追加保存無し)」**: 左端に配置された赤背景白地のボタン。現在のプロンプト内容を YAML ファイルに保存することなく、一時的に適用してチャットを開始する。
* **「保存して実行」**: 右端に配置された白枠黒地のボタン。下部にあるテキスト入力欄に入力された「プロンプト名」で現在のプロンプト内容を YAML ファイル（`prompts/prompts.yaml`）に保存し、チャットを開始する。プロンプト名が空欄の場合は簡単な警告を表示して処理を中断する。同名のプロンプト名が存在する場合は、上書き確認の確認用モーダルダイアログ（`st.dialog`）が表示され、ユーザーの同意を得たうえで上書き保存が実行される。
* **永続化ファイルの配置**: パッケージ外部（プロジェクトルート直下の `prompts/prompts.yaml`）に保存することで、wheel化した状態での実行時やアップデート時にもユーザーのカスタムプロンプトが失われないよう保護する。初回起動時には、パッケージ同梱のデフォルトが自動コピーされて配置される。

## 4. セッション管理とデータライフサイクル
* **セッションUUID**: 起動時に自動で一意のUUID（`session_uuid`）がセッションステートに割り当てられ、ディスク上の `temp_workspace/<UUID>/` に一時保存用ディレクトリが作成される。
* **自動履歴保存**: 会話が2往復以上続くと、自動的に `chat_log/` ディレクトリ配下に `YYMMDD_要約タイトル.json` というファイル名で会話履歴を自動保存（タイトルはAIが内容から自動抽出）。
* **会話の分岐（Branching）**: 
  過去のやり取りの特定の吹き出しの横にある「この会話から分岐」ボタンを押すと、その時点までのやり取りを保持したまま新しい履歴ファイル（`YYMMDD_タイトル-02.json` 等）を自動生成し、別チャットとして再開可能。

## 5. 動作前提条件とシステム要件
* **Pythonバージョン**: `Python >= 3.11`
* **動作OS**: Windows / macOS / Linux (PowerPoint/PDFエクスポートの一部機能はWindows OSかつEdge/Chrome/PowerPointのインストールが必要)
* **環境変数**: `./env/` 配下の `.env` ファイルに以下の環境変数が設定されていること。
  * `GCP_PROJECT_ID` (GCP プロジェクトID)
  * `GCP_LOCATION` (Vertex AI リージョン。例: us-central1)
  * `GEMINI_MODEL_ID` (デフォルトモデル。例: gemini-3.5-flash)
  * `AZURE_OPENAI_API_KEY` / `AZURE_OPENAI_ENDPOINT` (Azure接続用)
  * `AZURE_OPENAI_GPT54_DEPLOYMENT` (標準Azure用デプロイメント名)
  * `AZURE_OPENAI_CODEX_DEPLOYMENT` (Codex専用デプロイメント名。未指定時はGPT54用の設定を流用)
